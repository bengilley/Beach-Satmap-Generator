# Beach Satmap Generator — README пользователя

Упрощённая документация для использования генератора satmap через графический launcher.

Эта версия предназначена для обычного пользователя. В ней оставлены только сведения, нужные для установки, запуска, настройки и получения созданных файлов.

---

## 1. Для чего нужен этот инструмент?

Beach Satmap Generator создаёт более чистую satmap для карты DayZ, особенно в зонах пляжей, воды и береговой линии.

Из четырёх исходных файлов инструмент создаёт:

- финальную исправленную satmap;
- beach mask;
- версионную папку вывода, чтобы старые результаты не перезаписывались.

Инструмент рассчитан на большие карты, например `10240 x 10240 px`.

---

## 2. Необходимые файлы

Помести следующие файлы в одну папку:

```text
satmap_gui_launcher.pyw
satmap_generator_optimized_presets.py
input/
  heightmap.asc
  mask.png
  satmap.png
  layers.cfg
```

### Назначение файлов

| Файл | Назначение |
|---|---|
| `heightmap.asc` | Используется для определения высоты, воды, уклонов и береговых зон. |
| `mask.png` | Используется для распознавания текстур по цветам из `layers.cfg`. |
| `satmap.png` | Исходная спутниковая картинка, которая будет исправлена. |
| `layers.cfg` | Связывает цвета mask с названиями текстур/layers DayZ. |

Поддерживаемые форматы изображений: `PNG`, `JPG`, `JPEG`, `BMP`, `TIFF`.

Для `mask` рекомендуется использовать `PNG`, `BMP` или `TIFF`. Не рекомендуется использовать `JPG`, потому что он может изменить цвета mask.

## 3. Установка

### Простой способ через launcher

1. Запусти `satmap_gui_launcher.pyw`.
2. Открой вкладку **Files / Файлы**.
3. Нажми **Install dependencies / Установить зависимости**, если зависимости ещё не установлены.

### Ручной способ

Открой терминал в папке скрипта и выполни:

```bash
py -m pip install numpy pillow scipy
```

---

## 4. Запуск программы

Интерфейс можно запустить двумя способами:

```bash
py satmap_gui_launcher.pyw
```

или двойным кликом по файлу:

```text
satmap_gui_launcher.pyw
```

В интерфейсе есть четыре вкладки:

1. **Files / Файлы**
2. **Profiles / Профили**
3. **Technical / Техника**
4. **Run / Запуск**

---

## 5. Вкладка Files / Файлы

Эта вкладка используется для выбора файлов генератора.

Проверь особенно:

- путь к скрипту генератора;
- путь к heightmap;
- путь к mask;
- путь к satmap;
- путь к `layers.cfg`.

### Текстуры DayZ для распознавания

Нужно указать текстуры, которые скрипт должен использовать для распознавания пляжа, песка и, при необходимости, внутренней суши.

| Поле | Назначение |
|---|---|
| Текстура уже является пляжем | Текстура, которая уже считается пляжем или берегом. |
| Текстура песка для расширения | Исходная текстура, которую скрипт будет расширять вокруг берега. |
| Текстура земли для смешивания | Необязательно. Ограничивает смешивание песок → земля конкретной текстурой. |

Пример:

```text
Текстура уже является пляжем: hp_beach
Текстура песка для расширения: hp_sand
Текстура земли для смешивания: cp_grass
```

Если используется custom-текстура, введи её имя точно так же, как в `layers.cfg`.

## 6. Вкладка Profiles / Профили

Это главная вкладка для настройки внешнего вида.

### Рекомендуемые настройки для начала

| Настройка | Рекомендуемое значение |
|---|---|
| Пляж: размер и уклон | `4 - Wide beach` или `3 - Balanced` |
| Вода: уровни высоты | `1 - Normal water` |
| Смешивание песок → земля | `4 - Strong blend` или `3 - Natural blend` |
| Тип песка | `belle_ile` или `atlantic_light` |
| Тип воды | `atlantic_belle_ile` |
| Текстура песка | Необязательно |
| Текстура воды | Необязательно |

### Пляж: размер и уклон

Эти настройки определяют, где может создаваться песок.

| Параметр | Эффект |
|---|---|
| Максимальная ширина пляжа | Чем выше значение, тем дальше пляж может уходить от берега. |
| Разрешённый уклон | Чем ниже значение, тем лучше скрипт избегает склонов и обрывов. |
| Максимальная высота пляжа | Максимальная высота, на которой может создаваться песок. |

Совет: начни с `4 - Wide beach`, затем уменьши или увеличь значения, если песок уходит слишком далеко или его недостаточно.

### Вода: уровни высоты

Эти настройки определяют, какие зоны считаются водой или сушей по heightmap.

В большинстве случаев оставь:

```text
1 - Normal water
```

Используй более низкий или высокий профиль только если уровень воды плохо совпадает с твоей heightmap.

### Смешивание песок → земля

Эта настройка улучшает переход между пляжем и землёй.

| Профиль | Результат |
|---|---|
| Disabled | Без перехода внутрь суши. |
| Light blend | Короткий и мягкий переход. |
| Natural blend | Хороший компромисс. |
| Strong blend | Более заметный и чистый переход. |
| Dune effect | Более выраженный короткий эффект дюны. |

## 7. Цвета и текстуры

### Цвета песка

Полезные примеры:

| Preset | Использование |
|---|---|
| `belle_ile` | Натуральный светлый песок, хороший выбор по умолчанию. |
| `atlantic_light` | Светлый атлантический берег. |
| `golden` | Более золотистый песок. |
| `pale_white` | Очень светлый песок. |
| `grey_shell` | Серый/ракушечный песок. |
| `dark_volcanic` | Тёмный песок. |
| `red_ochre` | Красный/охристый песок. |

### Цвета воды

Полезные примеры:

| Preset | Использование |
|---|---|
| `atlantic_belle_ile` | Хороший выбор по умолчанию для атлантического берега. |
| `atlantic_open_ocean` | Более глубокая синяя вода. |
| `tropical_lagoon` | Светлая бирюзовая вода. |
| `mediterranean_blue` | Средиземноморский синий. |
| `fjord_dark` | Тёмная вода. |
| `muddy_water` | Мутная или илистая вода. |

### Дополнительные текстуры

Текстуры песка и воды добавляют визуальные детали. Они не меняют созданную зону.

| Текстура | Рекомендуемая интенсивность | Рекомендуемый масштаб |
|---|---:|---:|
| Песок | `0.30` до `0.60` | `1.0` |
| Вода | `0.15` до `0.35` | `1.0` |

## 8. Вкладка Technical / Техника

### Финальное разрешение

Для карты DayZ 10K используй:

```text
10240
```

### Память / скорость

Эта настройка управляет количеством строк, обрабатываемых за один chunk.

| Значение | Использование |
|---:|---|
| `512` / `1024` | Очень безопасно, но медленнее. |
| `2048` | Хороший компромисс. |
| `4096` | Рекомендуется при 32 или 64 ГБ RAM. |
| `8192` | Быстрее, но требует больше RAM. |

Для начала используй `2048`.

### Допуск цветов mask

Обычно оставь:

```text
0
```

Используй большее значение только если mask был сжат или изменён, и цвета больше не совпадают точно с `layers.cfg`.

### Диагностические изображения

Включай **Create diagnostic images / Создать диагностические изображения** только если нужно разобраться с проблемой генерации.

## 9. Генерация satmap

Во вкладке **Run / Запуск**:

1. Нажми **Check files / Проверить файлы**.
2. Исправь ошибки, если они есть.
3. Нажми **Start generation / Запустить генерацию**.
4. Дождись завершения процесса.
5. Забери файлы из папки `outputs`.

---

## 10. Созданные файлы

Результаты создаются в папке вида:

```text
outputs/output_V1/
outputs/output_V2/
outputs/output_V3/
```

Каждая генерация создаёт новую версию, чтобы не перезаписывать предыдущие результаты.

Главные файлы:

| Файл | Назначение |
|---|---|
| `satmap_final_10240.png` | Финальная satmap для проекта. |
| `beach_mask_10240.png` | Сгенерированный beach/water mask. |

Дополнительные файлы:

| Файл | Назначение |
|---|---|
| `generation_settings.json` | Сохранённые настройки генерации. |
| `COMPLETE_GENERATION_REPORT.md` или локализованный отчёт | Читаемый отчёт генерации. |
| `debug_masks/` | Диагностические изображения, только если включены. |

## 11. Частые проблемы

### Скрипт не находит мои текстуры

Убедись, что введённые имена точно совпадают с именами в `layers.cfg`.

Пример:

```text
hp_sand
```

не то же самое, что:

```text
hp sand
```

### Mask не совпадает с layers.cfg

Лучше использовать mask в `PNG`, `BMP` или `TIFF`.

Если mask в `JPG`, цвета могут измениться. В этом случае используй формат без потерь или немного увеличь допуск цвета.

### Песок уходит слишком далеко вглубь суши

Уменьши:

- максимальную ширину пляжа;
- максимальную высоту пляжа;
- разрешённый уклон;
- силу смешивания земли.

### Песка недостаточно

Постепенно увеличь:

- максимальную ширину пляжа;
- максимальную высоту пляжа;
- разрешённый уклон.

### Генерация идёт медленно

Это нормально для satmap 10K.

Чтобы ускорить работу:

- закрой тяжёлые программы;
- используй `chunk rows = 4096`, если PC стабилен;
- используй разрешение, подходящее для проекта.

### Программе не хватает RAM

Уменьши:

```text
Память / скорость
```

Попробуй `1024` или `2048`.

## 12. Использование через командную строку

Рекомендуемый способ использования — графический launcher.

Простая команда:

```bash
py satmap_generator_optimized_presets.py
```

Пример для карты 10K:

```bash
py satmap_generator_optimized_presets.py --target-size 10240 --chunk-rows 2048 --sand-preset 4
```

## 14. Важный совет

Всегда сначала сделай первую генерацию с рекомендованными настройками, прежде чем менять цвета, текстуры или продвинутые значения.

Так ты убедишься, что исходные файлы, mask и `layers.cfg` правильно распознаются.
